My take on learning Git and Github

September 15 2015

The difference between Git and Github is: Git is a version-control program that you download onto your computer. It lives locally on your computer and is run in Terminal. When you use terms like git init, git add . and git commit -m "message of changes implemented here", that is Git. Version control is especially awesome for coders because using git can be directly linked from the coding platform terminal thats on your local computer. Coders can share their work and can discuss changes before they are fully implemented on to their main work.

Git helps with tracking changes by allowing you to constantly make incremental changes to your code without messing up the original code written. Its kind of like adobe photoshop, where you draw a base and keep adding changes (layers) such as colours and shading without touching the original.Once you're happy with the drawing, you can merge all the layers to the original. Thats similarly what you can do with git.

Github is not on your computer but lives remotely in the Galaxy. The internet. You can store and access your work online. It helps to save your work and make improvements without changing the original code on the master branch that you're currently happy with. You can create multiple branches on the same code, write the changes made to the original, and keep making improvements in different repositories until satisfied enough to add to the original. Its a great way to make changes without actually changing the original code. That way you can keep track of the changes and not get any new bugs embedded in the original. You'll easily know where the bug is, in the new repository.
